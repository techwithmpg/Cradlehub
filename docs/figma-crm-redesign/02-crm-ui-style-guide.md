# CRM UI Style Guide

## Brand Feel

CradleHub CRM should feel like a premium spa operations system: calm, elegant, professional, and simple enough for non-technical front-desk staff during a busy day.

The interface should support fast scanning and confident action. It should not feel like a marketing site, a data warehouse, or a developer admin console.

## Color Direction

- Dark sidebar: Use a deep neutral or green-black sidebar to anchor the workspace and keep navigation stable.
- Cream/off-white background: Use a warm off-white main canvas so operational cards feel softer than plain gray.
- Forest green actions: Use forest green for primary actions, available states, confirmed work, and success.
- Warm gold accents: Use warm gold for premium brand accents, selected highlights, and subtle emphasis.
- Soft orange warnings only: Use orange for attention, pending setup, not checked in, and recoverable warnings.
- Red only for critical: Reserve red for blocked workflows, failed payment, cancelled/no-show, or urgent risk.

## Typography Direction

- Use clean SaaS typography with strong page titles and compact supporting text.
- Page titles should be confident and readable without being oversized.
- Body text should prioritize clarity for non-technical users.
- Labels should be compact, scannable, and consistent across tables, cards, drawers, and badges.
- Avoid dense paragraphs in the main work area. Use progressive disclosure for explanations.

## Layout Rules

- Put the main action above the fold on every page.
- Keep readiness warnings compact at the top of the page.
- Move detailed issue explanations into drawers, side panels, or expandable sections.
- Use a right rail for summaries, help, totals, selected item details, or context that supports the main work area.
- Use progressive disclosure for explanations, "how this works" content, and setup education.
- Keep tables and boards dense enough for daily operations, but with generous row hit areas.
- Avoid stacking too many large cards vertically before the operator reaches the core task.

## Card Rules

- Use soft borders and subtle shadows.
- Use rounded corners, but keep them restrained and professional.
- Keep spacing compact in operational pages.
- Avoid cards inside cards.
- Cards should represent distinct repeated items, summaries, empty states, or panels.
- Do not use decorative cards to frame entire page sections.

## Status Rules

- Green: available, confirmed, ready, completed successfully, paid.
- Orange: attention needed, pending review, not checked in, setup incomplete, recoverable warning.
- Red: critical blocker, failed payment, cancelled, no-show, missing required dispatch data.
- Muted: inactive, empty, off today, no data, informational.
- Blue or neutral: in progress, assigned, active trip, currently serving.

Status badges should use short labels and consistent shape. They should be readable in tables, cards, and mobile stacked layouts.

## Mobile Rules

- Stack sections vertically.
- Keep actions large enough for touch.
- Avoid horizontal overflow.
- Convert board columns to tabs, segmented sections, or accordion groups.
- Move right rail content below the main content or into drawers.
- Keep the primary action visible early.
- Use compact list rows rather than wide tables.
- Preserve all workflows, even when the layout changes.
